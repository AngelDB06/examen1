# Crea las ramas develop y feature/coding-test-docs

Crea la rama feature donde escribirá código de la aplicación, crea el archvio de las pruebas unitarias y un workflow que al hacer un push sobre la rama release se ejecutarán las pruebas, y si se pasan hace la documentación automática como artefacto preparada para descargar y visualizarla.

Posteriormente hará un merge en la rama develop cogiendo los archivos de feature.
