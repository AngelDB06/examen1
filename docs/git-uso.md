La configuración se ha realizado en el package.json para los scripts de test y documentación.

## Explicación del flujo
Las ramas de este proyecto se basan en un flujo GitFlow sencillo donde contamos con:

-/main (sólo para cuando el proyecto esté probado y correctamente documentado)
-/release (realizar pruebas unitarias y documentación)
-/develop (realizar cambios de código o desarrollarlo)
-/feature/coding-test-docs (realizar tareas concretas del proyecto)

Cada una utilizada para sus tareas.
